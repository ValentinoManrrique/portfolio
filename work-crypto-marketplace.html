01 // CONTEXT

A marketplace that outgrew its UI

The client ran a crypto marketplace — users could buy, sell, and trade digital assets, with a checkout flow that handled both fiat and crypto payments. The product had grown fast, but the design hadn't kept up.

Every screen was built in isolation. Buttons had five different shapes. Payment modals looked nothing like the rest of the product. The trading view and the e-commerce checkout spoke completely different visual languages.

The brief was clear: define a design system from the ground up — one that could unify the marketplace, the checkout, and the wallet interface without a full rebuild.

02 // PAYMENT_FLOW

Mapping the checkout complexity

Before touching a single component, I mapped every payment path — fiat, crypto, and mixed-cart flows. This exposed where the UX broke down and where the design system needed to carry the most weight.

04 // PROCESS

How it came together

01

Audit & Inventory

Catalogued every unique UI element across the product — found 40+ button variants, 12 different input styles, and zero consistent spacing logic.

02

Token Architecture

Defined the primitive → semantic → component token hierarchy. All color, spacing, radius, and typography decisions live in tokens, not hardcoded values.

03

Payment & Checkout Flows

Redesigned the checkout modal to handle fiat, crypto, and mixed-payment carts. Built clear transaction state feedback: pending, processing, confirmed, failed.

04

Component Library in Figma

Built every component with variants, states, and tokens wired — buttons, inputs, data tables, asset cards, wallet widgets, order rows, and payment modals.

05

Documentation & Handoff

Delivered annotated specs, usage guidelines, and a Figma component kit the dev team could implement incrementally without a full rewrite.

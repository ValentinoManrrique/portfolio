01 // CONTEXT

A publishing platform that needed a scalable design system

READZ is a platform that lets creators build interactive magazines and content experiences without code. As the product grew, the UI became inconsistent — different screens used different patterns, colors, and component styles.

I was brought in to audit the existing interface, build a design system, and create reusable templates the product team could extend without breaking consistency.

02 // WHAT_I_BUILT

A component library that scaled with the product

01

Design Audit

Catalogued all existing UI patterns across the editor, dashboard, and publishing flows. Identified 40+ inconsistencies.

02

Design Tokens

Established color, typography, spacing, and radius tokens in Figma. The foundation everything else is built on.

03

Core Components

Built 80+ components — buttons, forms, navigation, cards, modals — all with documented variants and states.

04

Page Templates

Assembled components into reusable page templates for the editor, viewer, and dashboard screens.

05

Ongoing Iteration

350+ hours of long-term engagement — designing new features, maintaining consistency, and supporting the dev team handoff.

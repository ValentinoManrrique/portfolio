// 01 — OVERVIEW

What you're building

The brief generator form on the portfolio sends 5 fields to an n8n webhook. n8n passes them to Claude, which generates a structured JSON brief. The brief comes back to the browser and renders in real time.

The result: every form submission becomes a lead log entry and a structured project brief — with zero manual work.

Time to complete

2–3 hours end to end. n8n setup is the longest part (~1.5hrs first time). Once the workflow is live, adding new nodes takes minutes.

// 02 — N8N SETUP

Set up n8n Cloud

01

Create account

Go to n8n.io → "Get started free". Free trial: 5 active workflows, 2,500 executions/month — plenty for this.

02

Add Anthropic credential

Settings → Credentials → Add → search "Anthropic" → paste your API key from console.anthropic.com. Name it Anthropic account.

03

Add Google Sheets credential (optional)

Credentials → Add → "Google Sheets" → follow the OAuth flow. Required only if you want lead logging.

// 03 — WORKFLOW

Build the workflow node by node

Node 1 — Webhook

New Workflow → Add node → Webhook. Configure:

→

HTTP Method: POST

Response Mode: Response Node — critical. This makes n8n wait for the Respond node before sending back the HTTP response.

→

Copy the production URL

Activate the workflow (toggle top-right). The production URL format is https://your-instance.app.n8n.cloud/webhook/[id]. Paste this into your site.

Node 2 — Basic LLM Chain

Add node → Basic LLM Chain → connect Anthropic credential → model: claude-haiku-4-5-20251001. Paste the prompt from section 04 into the User Message field.

Node 3 — Code (parse JSON)

See section 05 for the exact code.

Node 4 — Respond to Webhook

Respond With: JSON · Response Body: {{ $json }}

// 04 — AI PROMPT

The prompt (~110 tokens)

Paste this into the Basic LLM Chain User Message field. n8n uses {{ }} for variable interpolation from the webhook body.

Brief writer. JSON only, no markdown.

type: {{ $json.body.projectType }}
goal: {{ $json.body.goal }}
timeline: {{ $json.body.timeline }}
budget: {{ $json.body.budget }}
challenge: {{ $json.body.challenge }}

{"projectName":"3-5 word name","projectType":"{{ $json.body.projectType }}","scope":"2 sentences: what gets built and how it solves the challenge within the budget","phases":["4 phases as 'Name: deliverable' paced for the timeline"],"questions":"3 specific pre-kickoff questions\nseparated by newlines"}n8n — LLM Chain user message
Max tokens

Set Max Tokens to 1024 in the Anthropic node. Lower values truncate the JSON and cause parse errors.

// 05 — CODE NODE

Parse the AI response to JSON

The LLM returns a text string. This Code node parses it into a real JSON object before the Respond node sends it back.

const raw = $input.first().json.text.replace(/```json|```/g, '').trim()
try {
  return [{ json: JSON.parse(raw) }]
} catch(e) {
  // Fallback — prevents workflow failure on bad JSON
  return [{ json: { projectName: 'Project Brief', scope: raw.substring(0,300), phases: [], questions: '' } }]
}n8n — Code node
// 06 — DEPLOY

Go live

01

Activate workflow in n8n

Toggle to Active (top-right). The production webhook URL is now live.

02

Paste production URL in index.html

Find WEBHOOK_URL in the brief generator script and replace with your n8n production URL.

03

Push to Vercel

git push — Vercel auto-deploys. Test the form on the live site and check the n8n Executions tab.

CORS

n8n Cloud blocks requests from unknown origins. Add your Vercel domain in n8n Settings → Allowed Origins, or set * during testing.

// 07 — TROUBLESHOOTING

Common issues

!

Brief doesn't appear — loading forever

Workflow not Active, or Respond to Webhook not connected. Check n8n Executions tab for the failing node.

!

CORS error in console

Add your domain to n8n Settings → Allowed Origins.

!

JSON parse error

Max Tokens too low — the response is getting cut off. Set to 1024 in the Anthropic node.

!

500 error from webhook

Check the AI node — wrong model name or invalid/missing Anthropic API key.
